Nextcloud for Docker搭建和备忘📝

nextcloud 操作备忘

May 26, 2024   ·   1 min read

官方的搭建教程已经比较傻瓜，基本上使用docker compose一把梭就能起服务。

Redis

服务状态检查会提示使用redis获得更好的访问体验，如果很讨厌这个提醒，安装redis服务也很简单。在原有的docker-compose.yml中添加如下项目：

nextcloud:
  # 此处省略其他项
  depends_on:
    - redis  # 保证nextcloud在redis服务更改后重启
  environment:
    - REDIS_HOST=redis  # 默认redis，和你redis服务的name相同
    - REDIS_PORT=6379  # 默认端口号
    - REDIS_HOST_PASSWORD=  # 自行填入密码

redis:
  image: redis:latest  # 生产环境可以指定具体的tag，此处拉取最新
  container_name: redis
  restart: always
  command: redis-server --requirepass <password>  # 此处改成自定义密码，需和上面的密码一致

运行docker compose up -d重新部署后，nextcloud会自动检测并启用redis服务。

索引缺失

nextcloud在升级之后经常会遇到此问题，解决方法就是重新索引。运行docker exec -u www-data nextcloud php /var/www/html/occ db:add-missing-indices即可。

其他类似的occ maintenance:repair等使用occ的命令都可以类似方式执行。

其他配置问题

一般来说，和其他部署方式一样，修改相关的配置可以通过修改config.php文件的内容实现。此文件在映射挂载的相关目录中，虽然是Docker部署，但相当多的配置项并不支持通过Docker的环境变量进行预配置。不过修改config.php是高风险操作，建议做好备份测试。
