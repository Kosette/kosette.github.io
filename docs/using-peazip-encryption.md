## 使用peazip加密归档文件

[回到首页](./index.md)

总有一些私密的文件需要归档保存，但是不加密有总让人觉得不放心，更不必说可能需要上传到云端的文件。`peazip`提供了非常简便的加密方法，不管是zip还是7z都有对应的aes加密算法，至于具体的加密强度，有兴趣的可以查看`peazip`相关的文档说明。

加密文件需要提供加密密码和密钥文件（可选），这一点上和`keepass`数据库的使用方法几乎相同，鉴于安全性，密码可以选的长一点，使用各种密码生成器很方便获取，而密钥文件则可以使用`makefile`或者`dd if=/dev/urandom`等等方法生成一份随机的小文件，密码需要熟记，密钥文件也要有备份。一个经常容易搞错的事情就是为了所谓的安全性，把密码弄得很难记，密钥文件也是制作一份，这样一来就无法防范文件丢失的风险，加密文件只能防范文件内容泄露的风险，却不能够防范文件丢失的风险，而密码也好，密钥文件也好，都进一步加重了文件丢失风险，所以备份文件是非常必要的。

当然了，`peazip`加密文件还不够方便，尤其是需要配合云盘使用的时候，所以有需要配合云端使用的需求，可以权衡使用`cryptomator`，只是`cryptomator`需要安装虚拟磁盘，过于复杂（使用起来并不复杂）。
