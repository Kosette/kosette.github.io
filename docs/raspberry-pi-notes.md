### 安装 clash

原先在merlin系统上安装也是比较折腾的，当然使用一些修改版可能有现成的教程和方案，但我还是倾向安装merlin的原版系统，折腾的内容有时间也可以一并记录下来，防止忘记了。

到底是使用clash的主线版本还是使用premium版本，要考虑的无非是两点：1. premium版本只有闭源的预编译好的文件，开源还是闭源，这一点见仁见智，事实上即便是开源软件，大多时候还是使用开发者预编译好的文件； 2. premium版本支持TUN和rule-set。我选择了安装premium版本的clash，主要是为了使用rule-set，扩展性要比原本写死的yaml文件方便得多，如果有在使用surge/quantumultx之类的软件，也许更能体会到这一点。

下载好clash for ARMv7的文件后，传输到树莓派上，还需要准备两个文件Country.mmdb和config.yaml，前者去maxmind官网下载，或者让clash运行后自动生成，后者需要根据自己的需求按照模板自行修改。因为要为局域网内的其他机器做代理转发，所以开启了内置的dns服务，使用redir-host模式，当然我最早折腾的时候使用的是fake-ip模式，但是打游戏的时候延迟非常高，时不时的还连不上网，换成redir-host之后情况好了很多，避免了一些兼容性问题。这一次在树莓派上安装，我顺便尝试了TUN模式，但是能查找的关于TUN的资料很少，于是在不是很懂的情况下，使用查到的iptables代码，结果重启后发生了悲剧，ssh彻底连不上了，应该是网络设置坏了，于是又是一番折腾。开始我准备直接在tf卡上修改，为此还需要在windows10上安装ext2fsd软件，确实可以读取并修改ext4分区内的文件，但是改完之后上机没有效果，不知道什么原因，期间win电脑还因为不能正常卸载tf卡导致卡死机两次，可以说白费精力，没办法只能给树莓派重装系统。在此得到一个教训，如果没有其他方便连接到树莓派的方法，操作树莓派的网络要谨慎，在可以还原的情况下，先验证功能，这一波还是大意了，没有想到。

因为TUN模式开启后不知道怎么正确设置路由规则，所以最后还是放弃了。为了能够开机自启动，所以创建clash的systemd服务，具体可以查找到很多资料，以前是使用的cron脚本，当然merlin系统上有专门的user script来做这件事，对iptables设置，我没有永久保存，怕出问题后重启无效，所以还是用cron脚本来做，设置`@reboot`任务。

在运行clash的时候，如果直接`/bin/sh ./clash`，或者`chmod +x`赋予执行权限，软件会默认在`$HOME/.config/clash`下面生成或者调用config.yaml文件，但如果你不是root用户，而是使用sudo运行的话，这个目录会生成到root用户的`$HOME/.config`下，如果要使用其他目录，运行的时候加上`-d`参数。

### 安装 Samba 文件服务

安装samba还是非常简单的，网上可以搜索到很多教程。使用`apt install samba samba-common-bin`，接着`vim /etc/samba/smb.conf`编辑配置文件，如果需要从windows10访问，那么需要在[global]字段里面加入`min protocol = SMB2`，或者在win10的程序面板添加smb1.0支持，因为永恒之蓝漏洞的问题，win10默认关闭了smb1.0的支持。然后`smbpasswd -a <user>`输入用户密码，运行`service smbd restart`重启samba服务，就大功告成了。
